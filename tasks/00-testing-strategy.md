# Task 00: Testing Strategy

Status: active
Depends on: none
Spec refs: DESIGN-NEW.MD sections "11. Testing Strategy", "13. Failure Modes", "7. NBD Server"

Objective:
Define the V1 test pyramid and fixture policy before the NBD, file-ops, and
ask-bridge work lands, so the project does not drift back toward SQL-era
SQLite VFS smoke tests or depend on multi-GB real models for unit correctness.

Strategy:

- Unit tests use synthetic GGUF fixtures and direct byte buffers, not real models.
- Integration tests use tiny valid GGUF files copied into temp space before every mutating test.
- Real-model tests are opt-in via `LLMDB_E2E_GGUF=/path/to/model.gguf` and never part of the default fast path.
- NBD and `ask` E2E tests are opt-in; they require root/capabilities or a local `llama-server`.

Fixture policy:

- Primary test fixtures are synthetic GGUF v2 and v3 files generated by the test suite (already implemented in `tests/common/mod.rs`).
- Every destructive test clones a pristine fixture to a temp file and mutates the clone.
- Real GGUF files live in `/models/` (gitignored). A tiny real GGUF (e.g. stories15M) is preferred for local E2E; SmolLM3-3B is for manual runs only.

Test layers:

1. Unit
   - GGUF header and metadata parsing
   - Quant packer roundtrips and bit-mask invariants (one test per quant type)
   - Tensor eligibility classification (Skip / Tier1 / Tier2 / Lobotomy)
   - Tensor map address translation across tensor boundaries
   - Superblock and integrity block serialization with CRC32
   - Redirection table entry encode/decode
   - File table entry encode/decode

2. Integration (synthetic fixtures, `cargo test` default)
   - Device format/open: free list chained, integrity table zeroed, redirection is identity
   - Block roundtrip: write random block, read back, assert byte-for-byte
   - Shadow-copy crash simulation: abort between flush 1 and flush 2, reopen, verify old block intact and orphan block reclaimed
   - Integrity detection: flip a bit in the mmap, verify CRC32 failure on read
   - File store/get roundtrip: write a file, read it back, assert SHA256 equality
   - File ls and rm: verify tombstone + block chain returned to free list
   - Fill-and-free: allocate all blocks, free them, verify used_blocks returns to zero
   - Lobotomy mode: verifies embedding/norm/output tensors appear in tensor map only when enabled
   - NBD alignment: unaligned request (e.g. 3000-byte read at offset 100) returns correct bytes

3. Opt-in E2E (env-gated, `#[ignore]` by default)
   - Real GGUF roundtrip: `LLMDB_E2E_GGUF=...` — init, store file, get file, SHA256 match
   - NBD mount smoke: start NBD server, attach `/dev/nbdN`, format ext4, write a file via host FS, unmount, re-mount, verify file still there (requires root / `CAP_SYS_ADMIN`)
   - `ask` bridge: `LLMDB_E2E_GGUF=...` + local `llama-server` on $PATH — ask a question about a stored file, verify tool call dispatched and result returned

Benchmarks:

- Criterion benchmarks are in `benches/`, not in the test suite.
- Metrics: block read throughput, block write throughput, address translation rate.
- Perplexity and inference-latency at different storage utilizations are a Task 15 concern.

Acceptance criteria:

- Default `cargo test --offline` runs without downloading or mutating a real model.
- The synthetic fixture helper in `tests/common/mod.rs` supports both GGUF v2 and v3 (already true in V0 base; preserve through Task 02).
- Every quant packer has a roundtrip test that preserves non-stolen bits.
- `LLMDB_E2E_GGUF` is the only opt-in env var for real-model tests; NBD and ask tests gate on separate env vars (`LLMDB_E2E_NBD=1`, `LLMDB_E2E_ASK=1`).

# Task 00: Testing Strategy

Status: active
Depends on: none
Spec refs: `DESIGN.MD` sections "Implementation Plan", "Resolved Decisions", "Why ship this", "What We Cannot Recover From"

Objective:
Define the test pyramid and fixture policy before implementation work sprawls into multi-GB artifacts and brittle end-to-end-only coverage.

Strategy:

- Unit tests use generated synthetic fixtures and direct byte buffers, not real models.
- Integration tests use tiny valid GGUF files copied into temp space before every mutating test.
- Real-model tests are opt-in, local or nightly, and never part of the default fast path.
- Benchmark and quality runs use separate artifacts and are not treated as correctness tests.

Fixture policy:

- Primary test fixtures are synthetic GGUF v2 and v3 files generated by the test suite.
- Synthetic fixtures should include small tensor inventories with realistic names and quant layouts, but they do not need to be usable for inference.
- Every destructive test clones a pristine fixture to a temp file and mutates the clone.
- The repo should not depend on a multi-GB base model to pass unit or integration tests.

Test layers:

1. Unit
   - GGUF header and metadata parsing
   - Quant packer roundtrips and bit-mask invariants
   - Tensor eligibility, allocation ordering, and capacity math
   - Integrity metadata serialization and CRC validation
   - Recovery state transitions and shadow-write bookkeeping

2. Integration
   - `init`, `status`, `query`, `dump`, `load`, and `wipe`
   - SQLite VFS reads, writes, WAL behavior, and reopen recovery
   - Compression on/off paths
   - Lobotomy-mode allocation and diagnostics
   - Mixed-quant address translation through the stego device

3. Opt-in end-to-end
   - Real tokenizer extraction from an actual GGUF
   - `ask` session wiring against a local inference backend
   - KV-cache accounting
   - Perplexity and latency measurement across degradation levels

Real-model policy:

- Real-model tests require an explicit env var, for example `LLMDB_E2E_GGUF=/path/to/model.gguf`.
- Those tests are marked ignored by default.
- They always copy the source model into a temp location before mutation.
- A tiny real GGUF is preferred for local E2E checks. Large benchmark models are for manual or nightly runs only.

Acceptance criteria:

- Default `cargo test` runs without downloading or mutating a real model.
- The fixture strategy supports both GGUF v2 and v3.
- The repo has a clear separation between correctness tests and benchmark-quality experiments.
